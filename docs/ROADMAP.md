# 开发路线图 · Spoken

按比赛"每个 PR 只做一件事、粒度尽可能细、全周期持续交付"的要求，功能拆分为下列 PR 分步提交。每个 PR 标题以 `feat/fix/docs/chore` 前缀，描述包含：功能、实现思路、测试方式。

> 本路线图会随进度更新（勾选已合并的 PR）。

## PR 规范（自我约束）

- 标题：一句话说明本 PR 新增/修改了什么。
- 功能描述：作用与使用方式。
- 实现思路：技术选型或核心逻辑。
- 测试方式：如何验证该功能正常运行。
- 合并后 `main` 保持可运行。
